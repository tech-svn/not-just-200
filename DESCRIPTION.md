# Repository Description

## For GitHub Repo Settings

**Short Description (1 line for GitHub):**
```
Periodic website monitoring tool using headless Chromium - checks HTTP status, JavaScript execution, and all sub-resources (JS/CSS/images/API calls)
```

**Full Description (for README or About):**
```
Website Monitor - Simulates real browser behavior to monitor websites, not just HTTP status codes. Executes JavaScript, loads all resources, detects errors, sends Telegram alerts. Supports GitHub Actions automation and domain filtering.
```

## Usage in GitHub

1. Go to your repository → **Settings**
2. Scroll to **About** section
3. Click the gear icon (⚙️)
4. Enter:
   - **Description**: "Periodic website monitoring using headless Chromium - validates HTTP status, JavaScript execution, and all sub-resources"
   - **Website**: (optional) Your monitoring dashboard URL
   - **Topics**: `monitoring`, `playwright`, `website-health`, `github-actions`, `telegram`
